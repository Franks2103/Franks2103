## ⚙️ Engineering approach

I care about more than whether a system works during a demo.

I focus on:

- Clear responsibilities and data contracts
- Explicit failure modes, retries and fallbacks
- Idempotent and recoverable background processes
- Maintainable architecture over unnecessary complexity
- AI evaluation and human-in-the-loop workflows
- Observability across distributed components
- Measurable operational and business value

> Good engineering is not only making a system work. It is understanding how it behaves when things go wrong.
